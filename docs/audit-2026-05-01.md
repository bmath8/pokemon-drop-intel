# Audit Notes - 2026-05-01

## Scope

Reviewed the Next.js app foundation, API routes, parser/normalizer flow, dashboard state persistence, mailbox sync, webhook relay, README, package metadata, and verification commands.

Excluded generated output, `node_modules`, and unrelated sibling projects.

## Findings Addressed

- Fixed a local storage hydration race that could overwrite saved dashboard state, parsed history, mailbox filters, delivery settings, and product rules with defaults on first mount.
- Added explicit JSON error handling and input size limits for parser routes.
- Added upload size limits for raw email and mailbox archive parsing.
- Added default SSRF protection to webhook relay by blocking localhost and private-network destinations unless explicitly enabled with `POKEMON_ALLOW_PRIVATE_WEBHOOKS=true`.
- Updated global typography so the configured Geist font variables are actually used by the page body.
- Replaced a blocked Destined Rivals artwork URL with a stable official Pokemon asset after browser verification showed a 403 image failure.
- Converted product artwork rendering to `next/image` with narrow remote host/path allowlists so image lint warnings are resolved without broad external image access.
- Wrapped delivery controls in a form container to remove the browser warning for the optional password-style bearer token input.
- Replaced the default scaffold README with project-specific setup, verification, mailbox, webhook, and security documentation.

## Verification

- `npm run lint` passes.
- `npm run build` should be run with the dev server stopped on Windows if `.next` is locked by Next dev or OneDrive.
- `npm audit` reports two moderate advisories through Next's nested PostCSS dependency. The reported automatic fix suggests an invalid downgrade path, so this is documented rather than force-applied.

## Remaining Improvements

- Add unit tests for `email-parser`, `sku-normalizer`, and API guard behavior.
- Move webhook secrets out of local storage if the app becomes multi-user or remotely hosted.
- Add named delivery profiles and source scoring after enough session history exists.
